1. Run Comprehensive Performance Tests

Open the browser console and run:

window.performanceTests.run()

This will test:

  • Core Web Vitals (FCP, LCP, CLS, TTI)
  • Breakpoint detection
  • Motion preferences
  • Image lazy loading
  • Resource timing
  • Layout shift detection

2. Test Reduced Motion Support

Method 1: Chrome DevTools

  1. Press Cmd+Shift+P (Mac) or Ctrl+Shift+P (Windows/Linux)
  2. Type "Emulate CSS prefers-reduced-motion"
  3. Select "reduce"
  4. Reload the page

Expected behavior:

  • KPI numbers appear instantly (no counting animation)
  • Floating social icons don't animate
  • Hover effects are instant

Verify in console:

window.performanceTests.generateReport()

3. Test Lazy Loading

Open Network tab in DevTools:

  1. Filter by "Img"
  2. Reload the page
  3. Scroll down slowly

Expected behavior:

  • Hero images load immediately
  • Below-the-fold images load as you scroll
  • Images have loading="lazy" attribute

4. Test at Breakpoints

Test at exact breakpoint widths:

// Test mobile breakpoint
window.performanceTests.testBreakpoint(768)

// Test desktop breakpoint
window.performanceTests.testBreakpoint(1200)

Manual testing:

  • Resize browser to 768px width
  • Check layout transitions smoothly
  • Resize to 1200px width
  • Verify no layout breaks

5. Test Browser Compatibility

window.performanceTests.testBrowser()

Required browsers:

  • Chrome (latest 2 versions)
  • Firefox (latest 2 versions)
  • Safari (latest 2 versions)
  • Edge (latest 2 versions)
